üìã Standard Response Format

All API responses follow this consistent structure:

Success Response WITH Data:

{ "status": "success", "message": "Descriptive success message", "data": { // Actual response data (tweets, users, etc.) } }

Success Response WITHOUT Data:

{ "status": "success", "message": "Action completed successfully" // No 'data' field when there's nothing to return }

Error Response:

{ "status": "error", "message": "Error description" // No 'data' field in error responses }

Rules:

  • status: Always present ("success" or "error")
  • message: Always present (human-readable description)
  • data: Only included when there's actual data to return
  • Error responses never include data (except debug info in dev mode)
