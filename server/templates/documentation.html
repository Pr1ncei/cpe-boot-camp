üõ†Ô∏è Development Tips

üí° Pro Tips for API Testing:

  • Save your token: After login, copy the JWT token and use it in subsequent requests
  • Use tools: Postman, Insomnia, or httpie make API testing easier than curl
  • Check status codes: 200 = success, 401 = unauthorized, 404 = not found, 500 = server error
  • Reset data: Use POST /api/reset to restore original demo data anytime
  • Health check: Use GET /api/health to verify the API is running
  • Content-Type: Always include Content-Type: application/json for POST requests

üîß API Patterns & Best Practices

‚úÖ Authentication Flow

  1. Register or login to get JWT token
  2. Include token in Authorization: Bearer TOKEN header
  3. Token expires after 7 days - login again if needed
  4. Store token securely in your app (localStorage, etc.)

üì° HTTP Status Codes

  • 200 - Success
  • 201 - Created (new resource)
  • 400 - Bad Request (validation error)
  • 401 - Unauthorized (missing/invalid token)
  • 403 - Forbidden (access denied)
  • 404 - Not Found
  • 409 - Conflict (duplicate data)
  • 500 - Server Error

‚ö° Rate Limits & Performance

  • No rate limits currently implemented
  • Pagination: Use page & limit parameters
  • Caching: Tweet data updates in real-time
  • Performance: SQLite with indexes for fast queries

üîê Security Notes

  • HTTPS: Use HTTPS in production
  • CORS: Configured for all origins in development
  • Passwords: Bcrypt hashed (salt rounds: 10)
  • JWT: HS256 algorithm with secret key
  • Validation: Input sanitization on all endpoints
